## 阶段 1：数据库设计

### 1.1 ER 图 → 12 张表

```
Department(系) 1─N Major(专业) 1─N Class(班级) 1─N Student(学生)
Department 1─N Teacher(教师)
Teacher 1─N Topic(毕业设计题目)
Student 1─1 Selection(选题) N─1 Topic
Selection 1─N Submission(阶段文档) 1─N Review(审阅)
Deadline(流程期限) — 独立配置表
AuditLog(审计日志) — 独立记录表
SysUser — 管理员登录
```

### 1.2 表清单

| # | 表名 | 说明 |
|---|------|------|
| 1 | department | 系（院系ID, 名称） |
| 2 | major | 专业（专业ID, 名称, 所属系） |
| 3 | class | 班级（班级ID, 名称, 所属专业） |
| 4 | student | 学生（学号, 姓名, 密码, 班级, 联系方式） |
| 5 | teacher | 教师（工号, 姓名, 密码, 系, 研究方向, 联系方式） |
| 6 | topic | 毕业设计题目（题目ID, 名称, 描述, 导师, 状态） |
| 7 | selection | 选题记录（学生-题目 1:1 关联） |
| 8 | submission | 阶段文档（开题/中期/初稿/终稿） |
| 9 | review | 教师审阅评分 |
| 10 | deadline | 各阶段期限配置 |
| 11 | audit_log | 审计日志（按学期分区） |
| 12 | sys_user | 系统用户（管理员） |

### 1.3 SQL 脚本（按顺序编写）

1. **`01_schema.sql`** — 建表 + 主外键 + 检查约束 + 默认值
2. **`02_data.sql`** — 测试数据（2系×3专业×6班, 20教师, 30学生, 若干题目）
3. **`03_procedures.sql`** — 2 个存储过程：
   - `calc_final_score(student_id)` — 加权计算最终成绩（开题15%+中期15%+初稿30%+终稿40%）
   - `stat_dept_completion(dept_id)` — 统计某系毕业设计完成率
4. **`04_triggers.sql`** — 3 个触发器：
   - 选题后自动更新 `topic.status` 为「已选」
   - 文档提交时自动写入 `audit_log`
   - 成绩更新时自动重算综合分写入 `selection`
5. **`05_views.sql`** — 3 个视图：
   - `v_student_progress` — 学生进度总览
   - `v_topic_detail` — 选题详情（含师/生信息）
   - `v_score_ranking` — 成绩排名
6. **`06_indexes.sql`** — 索引创建 + EXPLAIN 性能对比：
   - 学号/工号唯一索引
   - `topic.title` 全文索引（与 LIKE 对比）
   - 高频查询复合索引
7. **`07_permissions.sql`** — 创建 3 个 MySQL 用户角色并 GRANT 最小权限
8. **`08_backup.sql`** — 完整备份/差异备份脚本 + 恢复流程
9. **`09_window_functions.sql`** — 窗口函数示例（RANK/DENSE_RANK/PERCENT_RANK）

## 8 项高阶数据库技术覆盖

| # | 技术 | 应用场景 |
|---|------|----------|
| 1 | 存储过程 | 加权计算最终成绩 + 统计系完成率 |
| 2 | 触发器 | 选题自动更新状态 + 文档提交通知 + 成绩重算 |
| 3 | 视图 | 进度总览 / 选题详情 / 成绩排名 |
| 4 | 索引优化 | 唯一索引 + 全文索引 + 复合索引 + EXPLAIN 对比 |
| 5 | 事务并发 | 选题场景 SELECT...FOR UPDATE，防止一题多选 |
| 6 | 窗口函数 | RANK/DENSE_RANK 学生排名，PERCENT_RANK 通过率 |
| 7 | 权限管理 | 3 个 MySQL 用户角色 + GRANT/REVOKE 最小权限 |
| 8 | 审计日志 | 触发器自动记录所有 DML（谁/何时/做什么） |
